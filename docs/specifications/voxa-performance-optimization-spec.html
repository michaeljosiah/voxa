1.Executive summary

Voxa's architecture is fundamentally sound: bounded channels with backpressure, a deadlock-safe data-loop/turn-worker split, system-frame priority routing, and streaming TTS in three of four providers. The remaining performance cost is concentrated in five places, all fixable without changing the public programming model:

  1. Per-frame heap churn in the hot loop. Every frame through every processor allocates a linked CancellationTokenSource (FrameProcessor.cs:139). Every inbound audio message is copied twice and allocated twice (WebSocketAudioSource.cs:65-68). Every VAD inference allocates two tensors and two arrays (SileroVadEngine.cs:55-72). At 50 audio frames/sec across an 8-stage pipeline this is thousands of avoidable Gen0 allocations per second per session, which translates directly into GC pauses — the worst possible failure mode for real-time audio.
  2. Reflection-based, triple-allocating wire serialization. Every outbound JSON envelope allocates an anonymous object, a JSON string, and a UTF-8 byte[], serialized via reflection (WireProtocol.cs:65-100, WebSocketAudioSink.cs:133). The fix — typed DTOs + JsonSerializerContext source generation + serializing straight to UTF-8 — also produces a versionable, documented contract.
  3. A send path that holds a lock across network I/O. All outbound traffic (audio, transcriptions, status) serializes through one SemaphoreSlim held during WebSocket.SendAsync (WebSocketAudioSink.cs:121-140). Replacing it with a single-writer outbound queue removes contention and gives us the queue we need to purge stale TTS audio on barge-in (ROADMAP P2).
  4. Provider TTFB left on the table. Azure TTS buffers the full synthesis before yielding the first byte; HTTP TTS providers pay a cold TLS handshake on the first sentence of every session; Azure STT copies every audio write (AzureSpeechToTextEngine.cs:52).
  5. Turn-detection latency dominates end-to-end latency. Per ROADMAP.md, the chained pipeline sits at ~1.6 s; 800 ms of that is VAD silence hangover and 300–500 ms is TTS first-byte. This spec attacks the controllable parts now (eager first-sentence flush, connection warmup, streaming Azure TTS, configurable hangover) and defines the seam for smart-turn classification.

Expected outcome: steady-state audio path with zero per-frame allocations in Voxa code, wire serialization cost reduced ~70–80% (1 allocation instead of 3–4, no reflection), voice-to-voice latency on the chained pipeline reduced from ~1.6 s toward the ~600 ms roadmap target, and barge-in that actually stops the bot's audio. Every change ships with a benchmark or allocation gate that proves it.

3.Architecture context

One pipeline per WebSocket connection. Frames flow through linked FrameProcessor nodes, each running two drain loops (an unbounded priority "system" channel and a bounded capacity-64 "data" channel). The chained (non-Realtime) composition from the sample server:

WS client ──binary PCM──▶ WebSocketAudioSource ─▶ SileroVadProcessor ─▶ SpeechToTextProcessor(Whisper/Azure) │ WS client ◀─binary PCM + JSON envelopes── WebSocketAudioSink ◀─ TextToSpeechProcessor ▲ SentenceAggregator ◀─ AgentLoopProcessor(IAgentTurnDriver/MAF) ◀─ TranscriptionFilter

Hot-path frequency assumptions used throughout this spec: client sends 20 ms PCM chunks at 16 kHz mono 16-bit (640 bytes, 50 msgs/sec); TTS yields 8 KB chunks at 24 kHz (~6 chunks/sec of speech); VAD runs 512-sample windows (~31 inferences/sec at 16 kHz, up to 62 at smaller windows). A "session" is one WebSocket connection.

WS1 — Core frame loop: eliminate per-frame CTS allocation Phase 1

Fixes: F-01Effort: ~0.5 dayRisk: medium (touches interruption semantics — read carefully)Project: src/Voxa.Core

Current behavior (FrameProcessor.cs:133-164): each data frame gets its own CancellationTokenSource.CreateLinkedTokenSource(ct) so an InterruptionFrame (handled on the system loop, line 120-125) can cancel exactly the in-flight frame. The allocation also registers a callback on the parent token — two costs per frame.

Key insight: interruptions are rare (a few per conversation); frames are constant (hundreds/sec). So reuse one linked CTS across frames and replace it only after it has been fired. The semantics that must be preserved exactly:

  • An interruption arriving while a frame is in flight cancels that frame's token.
  • An interruption arriving between frames cancels nothing (current behavior: _currentFrameCts is null in the finally). The next frame must run with a fresh, uncancelled token.
  • Processor shutdown (ct) still cancels everything.

Replace RunDataLoopAsync with:

src/Voxa.Core/Processors/FrameProcessor.cs — replacement for RunDataLoopAsync (lines 133-164)
private async Task RunDataLoopAsync(CancellationToken ct)
{
    // One linked CTS reused across frames; replaced only after an interruption fires it.
    // Interruptions are rare, frames are constant — this removes the dominant per-frame
    // allocation in the pipeline (one CTS + parent-token registration per frame).
    var frameCts = CancellationTokenSource.CreateLinkedTokenSource(ct);
    try
    {
        await foreach (var frame in _dataChannel.Reader.ReadAllAsync(ct))
        {
            // A system-loop interruption may have fired the shared CTS while no frame was
            // in flight (it cancels via _currentFrameCts only when non-null, but guard
            // against any cancelled leftover from the previous frame's interruption).
            if (frameCts.IsCancellationRequested)
            {
                frameCts.Dispose();
                frameCts = CancellationTokenSource.CreateLinkedTokenSource(ct);
            }

            _currentFrameCts = frameCts;
            try
            {
                switch (frame)
                {
                    case StartFrame s: await OnStartAsync(s, frameCts.Token).ConfigureAwait(false); break;
                    case EndFrame e: await OnEndAsync(e, frameCts.Token).ConfigureAwait(false); break;
                }
                await ProcessFrameAsync(frame, frameCts.Token).ConfigureAwait(false);
            }
            catch (OperationCanceledException) when (frameCts.IsCancellationRequested && !ct.IsCancellationRequested)
            {
                // Frame was preempted by an interruption — drop it and continue with a fresh CTS.
            }
            finally
            {
                _currentFrameCts = null;
            }

            if (frame is EndFrame) break;
        }
    }
    catch (OperationCanceledException) { /* shutdown */ }
    catch (ChannelClosedException) { /* shutdown */ }
    finally
    {
        frameCts.Dispose();
    }
}
Race analysis (do not skip): the system loop calls _currentFrameCts?.Cancel(). Three windows exist:
  1. Frame in flight — _currentFrameCts points at the shared CTS → cancelled → frame dropped, loop replaces the CTS at the top of the next iteration. Identical to today.
  2. Between frames — _currentFrameCts is null → no cancel. Identical to today.
  3. New window: system loop reads a non-null _currentFrameCts, the data loop finishes the frame and nulls it, then the system loop cancels the now-idle shared CTS. Today that cancel hit a disposed/finished per-frame CTS (no-op). With reuse, the next frame would start with a cancelled token — this is exactly what the IsCancellationRequested check at the top of the loop handles: it swaps in a fresh CTS before the frame runs. The check happens after the channel read, so no cancelled token can ever reach ProcessFrameAsync at iteration start. A cancel that lands after the check but before ProcessFrameAsync begins is indistinguishable from case 1 (frame legitimately preempted) — acceptable and matches Pipecat semantics.

WS2 — Transport receive path: single-copy audio, UTF-8 parsing Phase 1

Fixes: F-02, F-14Effort: ~0.5 dayRisk: lowProject: src/Voxa.Transports.WebSocket

Current behavior (WebSocketAudioSource.cs:35-90): every binary message goes receive-buffer → MemoryStream (copy 1) → ToArray() (copy 2 + alloc). Text messages go bytes → per-segment string → StringBuilder → final string → parse.

New behavior:

  • Binary fast path (a 640 B audio chunk always fits the 16 KB buffer in one receive): allocate the exact-size array once and copy once. The allocation is intentional — it becomes the AudioRawFrame.Pcm backing store with unbounded downstream lifetime, so pooling it is a correctness hazard (see §11).
  • Binary slow path (multi-receive message): accumulate in an ArrayPool-rented growable buffer, exact-size copy at end, return rented buffer in finally.
  • Text path: accumulate UTF-8 bytes (same pooled accumulator) and parse via the new UTF-8 overload WireProtocol.TryParseClientMessage(ReadOnlySpan<byte>) (added in WS3 — if implementing WS2 first, add the overload now delegating to JsonDocument.Parse which accepts UTF-8 directly; no string materialization).
src/Voxa.Transports.WebSocket/WebSocketAudioSource.cs — replacement for ReadLoopAsync
private async Task ReadLoopAsync(CancellationToken ct)
{
    var buffer = new byte[_options.ReadBufferSize];
    byte[]? acc = null;          // rented; only for messages spanning multiple receives
    int accLen = 0;

    try
    {
        while (!ct.IsCancellationRequested && _ws.State == WebSocketState.Open)
        {
            WebSocketReceiveResult result;
            try
            {
                result = await _ws.ReceiveAsync(new ArraySegment<byte>(buffer), ct).ConfigureAwait(false);
            }
            catch (OperationCanceledException) { return; }
            catch (WebSocketException ex)
            {
                await IngestAsync(
                    new ErrorFrame($"WebSocket receive failed: {ex.Message}", ex) { Direction = FrameDirection.Upstream },
                    ct).ConfigureAwait(false);
                return;
            }

            if (result.MessageType == WebSocketMessageType.Close)
            {
                await IngestAsync(new EndFrame(), ct).ConfigureAwait(false);
                return;
            }

            ReadOnlyMemory<byte> message;
            if (result.EndOfMessage && accLen == 0)
            {
                // Fast path — whole message arrived in one receive (the overwhelmingly
                // common case: 20 ms PCM chunks are ~640 B vs a 16 KB buffer).
                message = buffer.AsMemory(0, result.Count);
            }
            else
            {
                // Slow path — accumulate across receives in a pooled buffer.
                EnsureCapacity(ref acc, accLen + result.Count);
                buffer.AsSpan(0, result.Count).CopyTo(acc.AsSpan(accLen));
                accLen += result.Count;
                if (!result.EndOfMessage) continue;
                message = acc.AsMemory(0, accLen);
                accLen = 0;
            }

            if (result.MessageType == WebSocketMessageType.Binary)
            {
                // Single exact-size copy: this array is owned by the frame and flows
                // downstream with unbounded lifetime — do NOT pool it.
                var pcm = message.ToArray();
                await IngestAsync(
                    new AudioRawFrame(pcm, _options.InputSampleRate, _options.Channels),
                    ct).ConfigureAwait(false);
            }
            else // Text
            {
                var parsed = WireProtocol.TryParseClientMessage(message.Span);
                if (parsed is not null)
                {
                    await IngestAsync(parsed, ct).ConfigureAwait(false);
                }
            }
        }
    }
    finally
    {
        if (acc is not null) System.Buffers.ArrayPool<byte>.Shared.Return(acc);
    }

    static void EnsureCapacity(ref byte[]? acc, int needed)
    {
        if (acc is null)
        {
            acc = System.Buffers.ArrayPool<byte>.Shared.Rent(Math.Max(needed, 32 * 1024));
        }
        else if (acc.Length < needed)
        {
            var bigger = System.Buffers.ArrayPool<byte>.Shared.Rent(Math.Max(needed, acc.Length * 2));
            acc.AsSpan(0, acc.Length).CopyTo(bigger);
            System.Buffers.ArrayPool<byte>.Shared.Return(acc);
            acc = bigger;
        }
    }
}

WS4 — Send path: outbound queue, no lock-during-I/O, barge-in purge Phase 2

Fixes: F-04 + delivers ROADMAP P2's server halfEffort: ~1 dayRisk: medium (ordering + purge semantics)Project: src/Voxa.Transports.WebSocket

Design: replace the SemaphoreSlim with a bounded outbound channel drained by one writer task. Each queued message carries an epoch. An InterruptionFrame (which arrives on the processor's system loop, concurrent with the data loop — this is what makes the purge instant) bumps the epoch; the writer drops any audio message stamped with an older epoch. src/Voxa.Transports.WebSocket/WebSocketAudioSink.cs — full replacement
using System.Diagnostics;
using System.Text;
using System.Threading.Channels;
using Voxa.Frames;
using Voxa.Processors;
using Voxa.Transports.WebSocket.Protocol;

namespace Voxa.Transports.WebSocket;

/// <summary>
/// Pipeline sink that mirrors the downstream frame stream onto a WebSocket. AudioRawFrames are
/// sent as binary messages; everything else as JSON text per <see cref="WireProtocol"/>.
///
/// Outbound messages flow through a bounded single-writer queue: the data loop serializes and
/// enqueues (never blocking on network I/O), one writer task drains to the socket. On
/// <see cref="InterruptionFrame"/> the epoch is bumped and queued audio from the previous epoch
/// is dropped — the client hears the bot stop almost immediately (barge-in).
/// Caller owns the WebSocket's lifetime — this processor will not dispose it.
/// </summary>
public sealed class WebSocketAudioSink : PipelineSink
{
    private readonly record struct Outbound(ReadOnlyMemory<byte> Payload, bool IsBinaryAudio, int Epoch);

    private readonly System.Net.WebSockets.WebSocket _ws;
    private readonly Func<Frame, string?>? _customSerializer;
    private readonly Channel<Outbound> _outbound = Channel.CreateBounded<Outbound>(
        new BoundedChannelOptions(256)
        {
            FullMode = BoundedChannelFullMode.Wait,   // backpressure into the pipeline, never drop silently
            SingleReader = true,
        });

    private int _epoch;                  // bumped on interruption; writer drops older audio
    private Task? _writerTask;
    private long _turnT0;                // WS0.3: UserStoppedSpeaking timestamp for TTFB

    public WebSocketAudioSink(
        System.Net.WebSockets.WebSocket webSocket,
        Func<Frame, string?>? customSerializer = null)
        : base("WebSocketAudioSink")
    {
        _ws = webSocket ?? throw new ArgumentNullException(nameof(webSocket));
        _customSerializer = customSerializer;
    }

    protected override ValueTask OnStartAsync(StartFrame frame, CancellationToken ct)
    {
        _writerTask = Task.Run(() => WriteLoopAsync(ct), ct);
        return ValueTask.CompletedTask;
    }

    protected override async ValueTask ProcessFrameAsync(Frame frame, CancellationToken ct)
    {
        if (frame.Direction == FrameDirection.Upstream)
        {
            await base.ProcessFrameAsync(frame, ct).ConfigureAwait(false);
            return;
        }

        if (_ws.State == System.Net.WebSockets.WebSocketState.Open)
        {
            try
            {
                await EnqueueFrameAsync(frame, ct).ConfigureAwait(false);
            }
            catch (OperationCanceledException) { /* shutdown */ }
            catch (ChannelClosedException) { /* writer died with the socket */ }
        }

        await base.ProcessFrameAsync(frame, ct).ConfigureAwait(false);
    }

    private async ValueTask EnqueueFrameAsync(Frame frame, CancellationToken ct)
    {
        // Interruption is special: it arrives on the SYSTEM loop (concurrent with the data
        // loop), so the epoch bump takes effect immediately even while audio is queued.
        if (frame is InterruptionFrame)
        {
            Interlocked.Increment(ref _epoch);
            await WriteAsync(WireProtocol.BuildInterruption(), isBinaryAudio: false, ct).ConfigureAwait(false);
            return;
        }

        if (frame is UserStartedSpeakingFrame)
        {
            // Barge-in signal travels as a data frame in some compositions — bump epoch here
            // too so pipelines without an explicit InterruptionFrame still purge stale audio.
            Interlocked.Increment(ref _epoch);
        }

        if (frame is UserStoppedSpeakingFrame)
        {
            _turnT0 = Stopwatch.GetTimestamp();   // WS0.3
        }

        // Host-supplied custom serializer wins for any frame it claims (rare path; the
        // string→UTF8 encode here is acceptable).
        if (_customSerializer is not null)
        {
            var customJson = _customSerializer(frame);
            if (customJson is not null)
            {
                await WriteAsync(Encoding.UTF8.GetBytes(customJson), isBinaryAudio: false, ct).ConfigureAwait(false);
                return;
            }
        }

        switch (frame)
        {
            case AudioRawFrame audio:
                await WriteAsync(audio.Pcm, isBinaryAudio: true, ct).ConfigureAwait(false);
                break;
            case TranscriptionFrame t:
                await WriteAsync(WireProtocol.BuildTranscription(t), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case LlmTextChunkFrame chunk:
                await WriteAsync(WireProtocol.BuildText(chunk.Text), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case TextFrame txt:
                await WriteAsync(WireProtocol.BuildText(txt.Text), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case ToolCallRequestFrame call:
                await WriteAsync(WireProtocol.BuildToolCall(call), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case BotStartedSpeakingFrame:
                await WriteAsync(WireProtocol.BuildSpeaking("bot", started: true), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case BotStoppedSpeakingFrame:
                await WriteAsync(WireProtocol.BuildSpeaking("bot", started: false), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case UserStartedSpeakingFrame:
                await WriteAsync(WireProtocol.BuildSpeaking("user", started: true), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case UserStoppedSpeakingFrame:
                await WriteAsync(WireProtocol.BuildSpeaking("user", started: false), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case StatusFrame status:
                await WriteAsync(WireProtocol.BuildStatus(status.Message), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case ErrorFrame err:
                await WriteAsync(WireProtocol.BuildError(err.Message), isBinaryAudio: false, ct).ConfigureAwait(false);
                break;
            case EndFrame:
                await WriteAsync(WireProtocol.BuildEnd(), isBinaryAudio: false, ct).ConfigureAwait(false);
                _outbound.Writer.TryComplete();
                break;
        }
    }

    private ValueTask WriteAsync(ReadOnlyMemory<byte> payload, bool isBinaryAudio, CancellationToken ct)
    {
        VoxaMetrics.SinkQueueDepth.Record(_outbound.Reader.Count);
        return _outbound.Writer.WriteAsync(new Outbound(payload, isBinaryAudio, Volatile.Read(ref _epoch)), ct);
    }

    private async Task WriteLoopAsync(CancellationToken ct)
    {
        try
        {
            await foreach (var msg in _outbound.Reader.ReadAllAsync(ct))
            {
                // Barge-in purge: drop audio that was queued before the most recent interruption.
                if (msg.IsBinaryAudio && msg.Epoch != Volatile.Read(ref _epoch)) continue;

                if (_ws.State != System.Net.WebSockets.WebSocketState.Open) continue;

                if (msg.IsBinaryAudio && _turnT0 != 0)   // WS0.3: first bot audio after end of user speech
                {
                    VoxaMetrics.TurnTtfbMs.Record(Stopwatch.GetElapsedTime(_turnT0).TotalMilliseconds);
                    _turnT0 = 0;
                }

                try
                {
                    await _ws.SendAsync(
                        msg.Payload,
                        msg.IsBinaryAudio ? System.Net.WebSockets.WebSocketMessageType.Binary
                                          : System.Net.WebSockets.WebSocketMessageType.Text,
                        endOfMessage: true, ct).ConfigureAwait(false);
                }
                catch (System.Net.WebSockets.WebSocketException) { return; }  // connection died
            }
        }
        catch (OperationCanceledException) { /* shutdown */ }
        catch (ChannelClosedException) { /* shutdown */ }
    }

    protected override async ValueTask OnEndAsync(EndFrame frame, CancellationToken ct)
    {
        _outbound.Writer.TryComplete();
        if (_writerTask is not null)
        {
            try { await _writerTask.ConfigureAwait(false); } catch { /* best-effort drain */ }
        }
        await base.OnEndAsync(frame, ct).ConfigureAwait(false);
    }
}

WS6.6 — SentenceAggregator: incremental boundary tracking (F-11)

Replace the scan-everything-per-chunk logic (lines 39-70). Track the last boundary incrementally — each appended character is examined exactly once, and ToString happens only on flush, only for the flushed region:

src/Voxa.Speech.Abstractions/SentenceAggregator.cs — replacement for the LlmTextChunkFrame case + new fields
private readonly StringBuilder _buffer = new();
private readonly object _lock = new();
private int _lastBoundary = -1;   // index into _buffer of the last confirmed sentence boundary

// inside ProcessFrameAsync:
case LlmTextChunkFrame chunk when !string.IsNullOrEmpty(chunk.Text):
{
    string? toFlush = null;
    lock (_lock)
    {
        var text = chunk.Text;
        int baseLen = _buffer.Length;
        _buffer.Append(text);

        // Incremental scan: a boundary is . ! ? \n followed by whitespace (or end-of-buffer,
        // which we treat as boundary only once the NEXT char arrives and is whitespace — the
        // end-of-buffer case is handled below for the final char).
        // First, the char straddling the chunk seam: if the previous buffer ended with a
        // boundary char and this chunk starts with whitespace, that's a boundary.
        if (baseLen > 0 && char.IsWhiteSpace(text[0]) && IsBoundaryChar(_buffer[baseLen - 1]))
        {
            _lastBoundary = baseLen - 1;
        }
        for (int i = 0; i < text.Length - 1; i++)
        {
            if (IsBoundaryChar(text[i]) && char.IsWhiteSpace(text[i + 1]))
            {
                _lastBoundary = baseLen + i;
            }
        }
        // Trailing boundary char at end-of-buffer counts (matches old FindLastSentenceBoundary
        // semantics: "followed by whitespace OR end-of-input").
        if (IsBoundaryChar(text[^1]))
        {
            _lastBoundary = _buffer.Length - 1;
        }

        if (_lastBoundary >= 0)
        {
            toFlush = _buffer.ToString(0, _lastBoundary + 1).Trim();
            _buffer.Remove(0, _lastBoundary + 1);
            _lastBoundary = -1;
        }
        else if (_buffer.Length >= MaxBufferChars)
        {
            toFlush = _buffer.ToString().Trim();
            _buffer.Clear();
            _lastBoundary = -1;
        }
    }

    if (!string.IsNullOrEmpty(toFlush))
    {
        await PushFrameAsync(new TextFrame(toFlush), ct).ConfigureAwait(false);
    }
    return;
}

private static bool IsBoundaryChar(char c) => c is '.' or '!' or '?' or '\n';
Semantic difference to encode in tests: the old code re-scanned the whole buffer and could "un-confirm" nothing — but it found a trailing boundary char (end-of-input rule) on every chunk, so "Hello." followed by chunk " world" flushed "Hello." immediately on the first chunk. The incremental version preserves this (the text[^1] check). Also reset _lastBoundary = -1 in the interruption/OnEndAsync clear paths. Port the existing SentenceAggregator tests and add: boundary char as last char of chunk N + whitespace as first char of chunk N+1; "Mr." mid-sentence not splitting ("Mr. Smith" — note: '.' followed by space is a boundary under both old and new rules, so behavior is unchanged — copy expectations from existing tests, do not invent new linguistics).

11.Out of scope / deferred (with rationale)

  • Pooling AudioRawFrame payload buffers end-to-end. Frames fan out (sink + observability + capture processors can all hold the same frame) and have no disposal point, so pooled payloads need reference counting (IMemoryOwner + ref-count wrapper). The risk of a use-after-return audio glitch outweighs the remaining Gen0 win once WS1/WS2/WS5 land. Revisit if profiling after this spec still shows audio-buffer churn dominating; the design sketch is: AudioRawFrame gains an optional IMemoryOwner<byte> with a pipeline-sink-owned release, and every processor that retains a frame must Retain().
  • Frame record pooling / struct frames. Records are ~64–96 B Gen0 objects; after the CTS and buffer fixes they are a rounding error. Struct frames would break the type-switch programming model.
  • Smart-turn classifier implementations (OpenAI / local ONNX / HTTP) — separate ROADMAP P0 effort; WS8.1 delivers the seam they plug into.
  • Client-side (JS) playback flush on interruption — the other half of ROADMAP P2; server half (purge + envelope) ships in WS4. Verify and, if missing, fix in the demo client as a follow-up.
  • AllowSynchronousContinuations on pipeline channels. Tempting (~µs/hop) but lets one processor's code run inside another's write call, breaking the isolation that makes the interruption model reasoned-about. Do not enable without a dedicated analysis.
  • Deepgram/Cartesia providers — new packages per ROADMAP; orthogonal to this spec.
