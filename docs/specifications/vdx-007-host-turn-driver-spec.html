VDX-007 — Host-owned agent turns under UseDefaults()

A one-lookup composer seam: a host that registers its own IAgentTurnDriver keeps the composed defaults — real VAD, latency profile, diagnostics taps — while replacing the Microsoft-Agents stage entirely. The driver-level companion to VDX-006, for hosts whose agent is not an AIAgent at all.

Date 2026-07-08
Status Implemented
Target Voxa.AspNetCore (+ TranscriptionFilter, SileroVad hardening)

Depends on VDX-006 (shares the agent-stage seam discipline)

1. Problem

VDX-006 let a host own the voice agent's options (memory, frontend tools, hooks) under UseDefaults() — but only when the agent is a Microsoft-Agents AIAgent or an IChatClient. A host with its own agent runtime (Ada's IAdaEngine: hybrid local/cloud routing, its own memory, tool harness, approval gates) implements Voxa's IAgentTurnDriver directly and hand-builds an AgentLoopProcessor — which forces it off UseDefaults() for the entire chain. In practice (Ada M6.1) that meant silently losing the registered Silero VAD, the LowLatency profile, and the diagnostics taps, plus two local workaround stages (input-rate relabeling, blank-audio filtering) that belong upstream.

2. The seam

DefaultVoicePipelineComposer.CreateAgentProcessor resolves a host-registered IAgentTurnDriver from the per-connection scope before the existing AIAgent → IChatClient → IVoiceAgentFactory chain. When present:

  • The stage is new AgentLoopProcessor(hostDriver, maxResponseDuration: tuning.MaxResponseDuration) — the profile's response cap still applies.
  • The Microsoft-Agents wiring is bypassed, not layered: no AIAgent is required in DI, Voxa:Agent:* (provider, memory, instructions) does not apply, and a registered IVoiceAgentConfigurator is never invoked (a warning is logged if both are present).
  • With no driver registered the agent stage composes exactly as before — the byte-identity guarantee of VDX-006 is preserved.
// Host (Ada) — the whole migration:
services.AddSingleton<IAgentTurnDriver, AdaEngineTurnDriver>();
app.MapVoxaVoice("/voice").UseDefaults();
// VAD stop duration, aggregator chunking etc. stay config: Voxa:Vad:StopDurationMs = 900, …

3. Companion hardening (same PR)

ChangeWhy
TranscriptionFilter.DropBracketedMarkers (default true): drop transcripts fully enclosed in []/(). whisper.cpp emits [BLANK_AUDIO] on silence; the blocklists only caught known spellings. Retires Ada's custom BlankTranscriptionFilter stage.
Silero VAD rate-mismatch warning fires once per session (frames still forwarded without VAD). Mis-rated audio arrives at ~50 fps; a per-frame warning is a hot-path log flood for a session-wide wiring fault.
Non-goals. No wire, transport, or Core changes. The IAgentTurnDriver contract is untouched — this is purely a composer resolution-order addition. Voxa.Core stays zero-dependency.

4. Verification

  • ComposerHostTurnDriverTests: driver-only DI composes (previously threw); no driver ⇒ unchanged requirement; driver wins over IChatClient; driver + configurator warns and skips the configurator; pipeline shape unchanged.
  • TranscriptionFilterTests: bracket-enclosed markers dropped; embedded brackets pass; rule can be disabled.
  • SileroVadProcessorTests: three mis-rated frames ⇒ one warning, all forwarded.
  • Existing golden/route tests pin the no-driver path byte-identical.
