VLS-010 — Local on-device end-of-turn classifier (Smart Turn v3, ONNX)

The smart-turn seam (P0) shipped with remote implementations only — HttpSmartTurnClassifier and the Python sidecar. This spec fills the promised third slot from the seam's own doc comment: a local ONNX implementation, so the low-latency turn profile (VAD stop ≈ 200 ms + classifier confirm) works fully offline. The model is Smart Turn v3 — an openly published, BSD-2-Clause, multilingual semantic end-of-turn classifier, 8.3 MB, CPU-real-time.

The 8-second window semantics are pinned: keep-last-8 s, left-pad with zeros (speech at window end), normalized log-mel — locked by a short-utterance golden fixture.

1. Problem

ISmartTurnClassifier lets the VAD's StopDurationMs drop to ~200 ms without cutting off mid-sentence pauses: silence fires → classifier inspects the turn's audio → only a "complete" verdict emits UserStoppedSpeakingFrame. But both shipped implementations leave the device: an HTTP endpoint or a local Python sidecar process. That fails Voxa's local-speech story (Studio "never touches the network before the user acts"; Ada is private-by-default) and adds a network round-trip to the most latency-sensitive decision in the pipeline. Result: nobody runs the low-latency turn profile in practice — Ada ships a blunt 900 ms stop duration instead.

3. Design

3.1 Package shape

New leaf package Voxa.Audio.SmartTurn.Onnx, mirroring the Voxa.Audio.Diarization / Voxa.Audio.Diarization.Onnx split: the existing Voxa.Audio.SmartTurn keeps its zero-ONNX dependency surface (HTTP/sidecar); the .Onnx package references Voxa.Audio.Onnx (VLS-006 host — CPU by default, GPU execution providers opt-in with PrivateAssets discipline) and carries the catalog + classifier. 

public sealed class LocalSmartTurnClassifier : ISmartTurnClassifier
{
    // The ctor does NO eager work — no cache resolve, no session build, no download.
    // The first IsTurnCompleteAsync call lazily resolves the pinned model through
    // VoxaModelCache (honors VOXA_MODEL_CACHE), builds the shared OrtSession via the
    // VLS-006 host, and precomputes the mel filterbank. Composition stays cheap and
    // network-free; WarmUpOnStart opts hosts into eagerness (§3.3).
    public ValueTask<bool> IsTurnCompleteAsync(
        ReadOnlyMemory<byte> recentSpeechPcm, int sampleRate, CancellationToken ct);
}
  • Pipeline per call: PCM16 → float; resample to 16 kHz when needed (LinearResampler, Voxa.Audio.Abstractions); window to exactly 8 s with the reference semantics pinned: keep the last 8 s when longer, zero-pad at the beginning when shorter, so the speech always sits at the end of the window (right-padding shifts short utterances into positions the model never saw in training); normalized Whisper-style log-mel features (max-normalization enabled, matching the reference front-end); single session run; probability ≥ Threshold.
  • Failure posture: per the seam contract, any error (missing model, ONNX fault, timeout) returns complete and logs once — a classifier failure must never strand the conversation with an unclosable turn.
  • Concurrency: one session, inference serialized behind an async lock. Every caller gets a verdict computed on its own audio — concurrent calls (multiple sessions sharing the DI singleton, or a slow inference overlapping the next silence candidate) queue and run in turn; in-flight results are never shared across callers, since another session's audio would produce a meaningless verdict. Inference is single-digit ms, so queuing is invisible in practice; if a call has waited past its own deadline it resolves via the failure posture (complete) rather than adopting a stranger's answer. No per-call allocation of the feature buffers — pooled/reused, matching the data-loop allocation discipline.

3.2 Shared DSP: Whisper-style log-mel in Voxa.Audio.Onnx

The feature extractor is not classifier-specific — it is the standard Whisper front-end (n_fft = 400, hop = 160, 80 mel bins, Slaney mel scale, periodic Hann window, log10 with the usual max-normalization and clamp). VLS-007's ASR models (Parakeet/Nemotron families) need the same front-end, so it lands as WhisperLogMelFeatureExtractor in Voxa.Audio.Onnx (internal-visible-to friends or public-minimal — decide at review), with golden tests against precomputed reference feature frames checked in as small fixture arrays. The fixtures are generated with the reference front-end's normalized output (max-normalization on) and include a short-utterance case so the left-padding semantics of §3.1 are locked by test, not prose.

3.3 Registration & composition

// Host code — the default composer already wires any registered ISmartTurnClassifier
// into the VAD's ConfirmTurnEnd (P0), so this one line is the whole integration:
services.AddVoxaLocalSmartTurn();                       // defaults
services.AddVoxaLocalSmartTurn(o => { o.Threshold = 0.6f; o.WarmUpOnStart = false; });
  • Options: Threshold (default 0.5), MaxWindowSeconds (default 8), WarmUpOnStart (default false). With warm-up off, nothing happens until the first classification call (§3.1 lazy init); with it on, an ASP.NET hosted service resolves the cache and builds the session at startup so the first turn never pays the cold cost. Server hosts may accept an on-demand download at first use; Studio composes with RequireCached=true — the classifier never downloads implicitly there; a missing model logs once and fails "complete" until the user downloads it through the Config card (§3.3 below), preserving the explicit-consent download pattern.
  • Config-first activation for the AddVoxa registry path: Voxa:SmartTurn:Provider = "local" (alongside the existing http/sidecar providers), validated fail-fast on unknown values.
  • With VRT-006: descriptor participates in TurnEnd=Auto preference (engine EOT > smart-turn > silence).
  • Studio: the model appears in Config as a downloadable artifact (the pyannote pattern — size + license shown, explicit user-initiated download, cache-path honesty), and the Builder's VAD node can select it where registered.

4. Non-goals

  • Streaming/continuous turn scoring (scoring every frame instead of once per silence event). The seam is verdict-at-silence; continuous scoring is a different seam and a VRT-006 follow-up if ever needed.
  • GPU-required operation. CPU is the default and the target; GPU EPs come free via the VLS-006 host but are never required.
  • Model training/fine-tuning, quantization work, or alternative EOT models. One pinned artifact; the catalog pattern makes additions cheap later.
  • CoreML/mobile variants. Windows/Linux/macOS CPU via ONNX Runtime only.

5. Workstreams

  • WS1 — DSP: WhisperLogMelFeatureExtractor in Voxa.Audio.Onnx + golden feature-frame tests (offline, default suite).
  • WS2 — Classifier: package, catalog, LocalSmartTurnClassifier, options, registration extensions, registry provider id local; unit tests with a stub OrtSession seam for the failure-posture, pooling, lazy-init (ctor does nothing; first call initializes once under concurrency), and per-caller verdict isolation (two overlapping calls with different audio get different, correct answers).
  • WS3 — Integration: [Trait("Category","LocalModels")] tests: real download → hash verify → inference on checked-in fixture utterances (a complete sentence ⇒ complete; an obvious mid-sentence pause ⇒ incomplete); cache-hit offline re-run.
  • WS4 — Surfacing: Studio Config download card + provider selection; docs (docs/local-speech.md gets an end-of-turn section, performance-tuning low-latency profile updated to reference it), CHANGELOG.

6. Verification

  • Default suite: no network, no model — DSP golden tests + stub-session tests only.
  • LocalModels lane: pinned download resolves and verifies; both fixture utterances classify correctly at default threshold; second run with network blocked passes (cache).
  • Latency budget: inference ≤ 25 ms p99 on the CI CPU (generous bound — it's the turn-taking path); recorded via the existing stage-latency histogram under a voxa.stage.latency smart-turn stage tag.
  • Golden composition: unregistered ⇒ byte-identical pipelines (existing test unchanged).
  • Live protocol: Studio Talk with StopDurationMs=250 + local classifier — pause mid-sentence ("my favourite colour is … hmm … probably blue") and confirm the turn holds; automated once VDX-009 lands.
