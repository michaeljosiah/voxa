Voxa sample

Pick a route, click Start. Mic streams to the server; assistant audio plays back. Server-side JSON envelopes show in the log.

idle
How does this work?

The browser captures mic audio via getUserMedia + an AudioWorklet, resamples to the route's target rate (24 kHz for Voice Live, 16 kHz for granular STT), converts Float32 → Int16, and ships it as binary WebSocket frames. Incoming binary frames are raw PCM played back via the Web Audio scheduler.

If you see missing config in the log, set the relevant secret with dotnet user-secrets set "Section:Key" "value" and restart the server.
